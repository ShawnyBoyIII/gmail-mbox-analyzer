## 2026-06-12 - Explicit Copy to Clipboard Button in Search Query Popup
**Learning:** For user interfaces that generate text specifically intended to be copied and pasted elsewhere (like a search query string), relying solely on auto-highlighting is insufficient. Explicit, one-click "Copy to Clipboard" buttons provide clear affordance and significantly reduce interaction friction. Providing a temporary visual confirmation (e.g., changing the button text to "Copied!") reassures the user that the action succeeded. When dealing with temporary feedbacks scheduled via Tkinter's `after()`, checking `widget.winfo_exists()` is a crucial pattern to avoid unhandled exceptions if the user closes the window before the feedback is reverted.
**Action:** Always provide explicit, one-click "Copy to Clipboard" buttons alongside any copy-paste content in the GUI, featuring temporary visual confirmation. And always use `winfo_exists()` in `after()` callbacks that revert visual feedback.
